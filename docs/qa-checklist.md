# TravelerLocal.ai — 发布前 QA 核心流程清单

> 每次发布前必须手动验证以下流程。全部通过才可上线。
> 测试环境：生产预览 URL 或 `http://localhost:3000`

---

## ✅ 检查清单

### 1. 首页加载

- [ ] 页面在 3 秒内完成加载，无白屏、无 JS 报错
- [ ] 底部导航显示 6 个 tab，顺序正确：**Home / Plan / Food / Explore / Essentials / Profile**
- [ ] 每个 tab 图标和文字均正常显示（🏠 🗺️ 🍜 🔍 🛡️ 👤）
- [ ] 默认激活 Home tab（teal 高亮 + 底部小圆点）

---

### 2. Chat 功能

- [ ] 点击 Home tab，进入 Chat 界面
- [ ] 输入框可正常输入英文
- [ ] 发送消息："What should I do in Beijing?"
- [ ] AI 在 10 秒内以**英文**回复（不是中文）
- [ ] 回复内容与北京相关
- [ ] 消息气泡正常显示，滚动流畅

---

### 3. Food 页面

- [ ] 点击 Food tab（🍜），页面正常跳转
- [ ] 页面有内容展示（菜品列表、分类或推荐）
- [ ] 无空白页、无 loading 卡死
- [ ] 图片/图标正常加载

---

### 4. Plan 功能

- [ ] 点击 Plan tab（🗺️），进入行程规划页
- [ ] 输入："3-day Beijing itinerary"
- [ ] 提交后 AI 返回**北京**行程（不是上海或其他城市）
- [ ] 行程包含 3 天内容，结构清晰
- [ ] 无报错、无超时

---

### 5. Profile — Guest 状态

- [ ] 未登录时点击 Profile tab（👤）
- [ ] 页面显示 **"Sign In"** 按钮
- [ ] 页面显示 **"Create Account"** 按钮
- [ ] 不显示用户信息或已登录内容

---

### 6. Google 登录

- [ ] 点击 "Sign In with Google" 按钮
- [ ] 浏览器跳转至 Google OAuth 授权页（`accounts.google.com`）
- [ ] URL 包含正确的 `client_id` 和 `redirect_uri`
- [ ] 未出现 OAuth 错误页

---

### 7. 语言切换

- [ ] 找到语言切换入口（Profile 页或设置）
- [ ] 切换到**中文**
- [ ] 底部导航文字变为中文：**首页 / 计划 / 美食 / 探索 / 必备 / 我的**（或对应译文）
- [ ] 页面主要文字随之变为中文
- [ ] 切换回英文后恢复正常

---

### 8. Essentials 页面

- [ ] 点击 Essentials tab（🛡️），页面正常加载
- [ ] 显示中国紧急联系电话：
  - [ ] **110**（警察）
  - [ ] **119**（消防）
  - [ ] **120**（急救）
- [ ] 联系信息清晰可读，无遮挡

---

### 9. 底部导航顺序验证

- [ ] 从左到右依次为：**Home → Plan → Food → Explore → Essentials → Profile**
- [ ] 点击每个 tab 均能正确跳转对应页面
- [ ] 当前激活 tab 高亮显示（teal 色）
- [ ] 非激活 tab 为灰色
